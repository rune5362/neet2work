# 채용 사이트 상세 공고 샘플 감사

확인일: 2026-05-13

## 목적

목록 수집은 가능하지만 상세 공고 필드가 아직 불확실한 사이트를 한국/일본 구분 없이 다시 확인한다. 대량 수집이 아니라 공개 상세 URL 소량 샘플 기준으로, 크롤러 구현 전에 어떤 상세 필드가 실제로 보이는지 판단한다.

## 확인 원칙

- 로그인, 캡차, 유료 기능, 비공개 API 우회는 하지 않는다.
- 공개 검색 결과 또는 공개 상세 URL에서 보이는 텍스트만 기준으로 본다.
- 사이트별 샘플은 현재 시점의 일부 공고이므로, 구현 전 selector 안정성은 다시 확인한다.
- 상세 본문이 이미지 또는 로그인 영역에 묶인 경우에는 수집 가능 필드와 한계를 분리한다.

## 사이트별 상세 확인

### 1. 잡코리아

- 샘플 URL: https://www.jobkorea.co.kr/Recruit/GI_Read/47235768
- 샘플 공고: 와탭랩스 백엔드 개발자
- 접근 결과: 공개 상세 URL 접근 가능

**확인된 상세 필드**

- 회사명
- 공고 제목
- 합격축하금
- 사원수
- 기업구분
- 산업/업종
- 회사 위치
- 복지/회사 태그 일부

**부족한 부분**

- 직무 설명, 주요업무, 자격요건, 우대사항 같은 핵심 상세 본문은 공개 텍스트 fetch 기준으로 안정적으로 보이지 않음
- 실제 상세 본문이 이미지, 동적 렌더링, 별도 영역으로 분리되어 있을 가능성이 있음

**판단**

잡코리아는 1차 크롤러에서 목록 중심으로 두는 편이 낫다. 상세 수집은 회사정보 보강 정도로 쓰고, 직무 상세 분석용 핵심 소스로 잡기에는 아직 약하다.

### 구현 메모

- 일본 종합 3개 사이트는 상세 본문이 풍부하므로 먼저 구현할 가치가 있다.
- Green, CareerCross, Daijob은 글로벌/개발자/언어 조건 필드가 좋아 포트폴리오 차별화에 유리하다.
- 잡플래닛과 Indeed는 한국 상세 보강용으로 좋지만, 중복/외부 출처/공고별 편차를 반드시 처리해야 한다.
- 잡코리아는 당장은 목록 수집을 중심으로 구현하고, 상세는 회사 정보 보강 필드로 제한한다.
