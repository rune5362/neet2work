# 일본 채용 사이트 수집 가능 필드 감사

확인일: 2026-05-13

## 목적

국내 채용공고 수집기 이후 글로벌 확장성을 보여주기 위해 일본 채용 사이트 후보를 정리한다. LinkedIn, Indeed처럼 약관/차단 리스크가 큰 글로벌 플랫폼을 무리하게 긁기보다, 일본 내 공개 채용 사이트와 한국인 대상 일본 취업 플랫폼을 1차 확장 후보로 검토한다.

## 공통 판단 기준

| 항목 | 확인 내용 |
| --- | --- |
| 목록 접근 | 로그인 없이 채용 목록 또는 공고 카드가 보이는지 |
| 상세 접근 | 로그인 없이 상세 공고 또는 상세급 데이터가 보이는지 |
| 목록 필드 | 회사명, 공고 제목, 지역, 직무, 마감일, 급여 등 목록에서 바로 얻을 수 있는 값 |
| 상세 필드 | 직무 설명, 자격요건, 언어 조건, 급여, 근무조건, 복지, 기업정보 등 상세에서 얻을 수 있는 값 |
| 수집 방식 | HTTP + HTML 파싱, 렌더링 후 API 확인, 공식 API 후보, Playwright fallback 필요 여부 |
| 리스크 | 로그인, 내부 API 의존, JS 렌더링, 약관 미확인, 중복 공고, selector 불안정성 |
| 포트폴리오 가치 | Neet2Work의 자기소개서/공고 적합도 분석과 글로벌 취업 확장 스토리에 맞는지 |

## 판단 등급 기준

| 등급 | 의미 |
| --- | --- |
| 1차 후보 확정 | 일본 확장 수집 후보로 유지하고, 약관/상세 구조 확인 후 구현 검토 |
| 조건부 우선 | 데이터는 좋지만 내부 API, 로그인, 카테고리 필터 등 추가 확인이 필요함 |

| 보류 | 공식 API 자격이 필요하거나 공개 목록 데이터가 부족해 현재 자동 수집 후보에서 제외 |

## 사이트별 감사

### 1. KOREC

- URL: https://app.korec.kr/job-hunting
- 성격: 한국인 대상 일본 취업 플랫폼
- 목록 접근: 가능
- 상세 접근: 조건부 가능
- 확인 방식:
  - 로컬 Chrome 실행 파일을 Playwright로 구동해 렌더링 후 확인
  - `chrome@openai-bundled` 플러그인은 현재 세션 도구로 직접 노출되지 않아 대체 방식 사용
- 목록에서 확보 가능한 필드:
  - 공고 수 70건
  - 회사명
  - 공고 제목
  - 마감일
  - 모집 분야
  - 입사 시기
  - 월급
  - 상장 여부 일부
  - 한국어/일본어 혼합 공고 텍스트
- 상세급 데이터에서 확보 가능한 필드:
  - 공고 ID
  - 회사명
  - 공고 제목
  - 근무 지역
  - 근무지 주소
  - 계약 형태
  - 전공 조건
  - 예상 급여
  - 월급 상세
  - 일본어 조건
  - 영어 조건
  - 직무 상세
  - 인재상
  - 복지
  - 상여
  - 주거 수당
  - 근무 시간
  - 연차
  - 교육 프로그램
  - 마감일
  - 입사 시기
  - 공고 상태
- 수집 방식 후보:
  - 1차: 렌더링 후 API 구조 분석
  - 앱은 `https://pageapi-hn7w3foona-dt.a.run.app/student-jobs`를 호출함
  - 브라우저 요청에는 `x-project-id: korec-stage` 헤더가 포함됨
- 리스크:
  - 첫 공고 카드를 클릭하면 상세 페이지 이동 대신 로그인 모달이 노출됨
  - 직접 API 호출은 브라우저 요청 헤더 없이 `500 Failed to fetch student jobs`를 반환함
  - 공식 API가 아니라 앱 내부 API에 가까우므로 약관 확인 전 자동 수집 확정은 위험함
- 1차 구현 여부: 조건부 우선
- 판단 이유:
  - Neet2Work의 “한국인 취업 준비자 → 일본 취업 확장” 스토리와 가장 잘 맞음
  - 자기소개서 분석에 쓸 수 있는 직무 상세, 인재상, 언어 조건, 급여 조건이 풍부함

## 데이터 모델 확장 메모

일본/글로벌 소스까지 고려하면 기존 `JobPosting` 외에 아래 필드가 필요하다.

- `country`: KR, JP 등 국가 코드
- `language`: ko, ja, en 등 원문 언어
- `source`: 사이트 키
- `sourceJobId`: 사이트 내 공고 ID 또는 URL hash
- `salary`: 연봉/월급/시급
- `currency`: KRW, JPY 등 통화
- `languageRequirements`: 일본어/영어 등 언어 조건
- `visaSupport`: 비자 지원 여부
- `remoteOption`: 리모트/하이브리드/온사이트
- `employmentType`: 정규직, 계약직, 인턴 등
- `expectedStartDate`: 입사 시기
- `raw`: 원본 HTML/JSON 일부 저장용
